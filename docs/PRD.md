# 產品需求文件 (PRD) - 智慧食譜管理系統

## 1. 專案概述

### 背景與動機
現代人生活忙碌，有時難以靈感地準備三餐，或者收藏了許多食譜卻難以系統化地管理。現有的筆記軟體或部分網站過於零散，缺乏統一且針對料理需求（如步驟圖、食材份量）設計的介面。因此，建立一個「智慧食譜管理系統」，幫助使用者輕鬆建立、分類及瀏覽食譜，並以圖文並茂的方式引導烹飪過程。

### 目標用戶
- 喜愛自炊的家庭主管／主夫。
- 想要學習烹飪的新手，需要清晰分步引導與圖片輔助。
- 熱愛記錄自己獨家配方，並希望透過系統歸納整理的料理愛好者。

### 核心價值主張
提供直覺且專為「料理步驟」打造的記錄與瀏覽體驗。讓食材清單與步驟一目了然，隨時隨地能開啟食譜跟著做，提升烹飪效率與成功率。

## 2. 功能需求

### 1. 新增食譜與基本資訊
- **描述**：使用者可建立新食譜，包含食譜名稱、預覽圖面與預估烹飪時間。
- **使用者故事**：作為一名 **料理愛好者**，我希望 **能新增並設定食譜的名稱、首圖和所需時間**，以便 **我能快速在列表中辨識並評估是否要在今天烹調該道菜**。

### 2. 清單式材料管理
- **描述**：提供結構化的食材清單，能清楚列出食材名稱及對應的克數或份量。
- **使用者故事**：作為一名 **負責備料的人**，我希望 **能以清單方式查看所有需要的食材與精確克數**，以便 **去超市採購與在家備料時不會遺漏或抓錯份量**。

### 3. 分步引導（烹飪步驟）
- **描述**：提供按步驟 1、2、3 依序排列的做法說明。
- **使用者故事**：作為一名 **烹飪新手**，我希望 **系統能將做法拆解成明確的先後步驟**，以便 **我能按部就班地完成料理，避免手忙腳亂**。

### 4. 圖片格式化支持（步驟圖）
- **描述**：每個烹飪步驟都允許上傳或附上一張圖片。
- **使用者故事**：作為一名 **視覺學習者**，我希望 **每個步驟都能配上一張對應的圖片**，以便 **能比對鍋中食物的狀態是否與教學一致，提高成功率**。

### 5. 分類標籤
- **描述**：支援為食譜加上如「中式」、「西式」、「甜點」等標籤分類。
- **使用者故事**：作為一名 **需要決定今晚菜色的使用者**，我希望 **能用分類標籤（如中式或西式）來過濾食譜**，以便 **快速依據當天想吃的口味找到合適的料理**。

## 3. 非功能需求

### 技術限制
- **前端與模板**：使用 Jinja2 結合 HTML/CSS 渲染頁面。
- **後端框架**：使用 Python Flask 建構。
- **資料庫**：使用 SQLite 進行輕量級資料儲存。

### 效能與安全考量
- **圖片載入**：預覽圖與步驟圖需注意優化或適當縮放，確保帶有多圖的食譜頁面仍能快速載入。
- **資料防護**：防範基礎的 SQL 注入攻擊與 XSS 攻擊（Flask/Jinja2 預設有跳脫防護）。處理使用者上傳的圖片時須驗證檔案類型與大小。
- **RWD 響應式設計**：介面需適合在手機平板上瀏覽，方便使用者在廚房邊看邊做菜。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須擁有)
- 新增與儲存食譜的基本資訊（名稱、首圖、時間）。
- 輸入與顯示結構化的食材清單與份量。
- 輸入與顯示有序的烹飪步驟（文字描述）。
- 依分類標籤瀏覽食譜清單。

### Should Have (應該擁有)
- 單一步驟附加一至多張圖片的功能。
- 首頁顯示全部食譜縮圖與名稱。
- 簡易的編輯與刪除食譜功能。

### Nice to Have (最好擁有)
- 搜尋功能（依關鍵字搜尋食材或食譜名）。
- 食譜份量自動換算（如：2人份自動轉換成4人份）。
- 烹飪時間倒數計時器整合至詳細頁面中。
